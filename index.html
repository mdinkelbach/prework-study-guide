HTML

  • The "head" element contains information about the webpage. This information will not be seen.
  • The "body" element represents the visible content shown to the user.
  • The "header" element within the "body" element is different than the "head" element.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A '.{variable}' indicates a class selction of a specific variable, causing the changes to only affect sections with that class.
  • The CSS file is linked within the "head" element.

Git

  • git init: used to initilize a fold as "main", followed by an add and commit (initial commit)
  • git remote add [origin {github link}]: used after initialization, adding, and commiting; done with the Github link as. Followed by git push -u origin main.
  • git clone [url]: obtain a repository from an existing URL
  • git status: tells you which branch you are currently on
  • git pull [origin {branch}]: takes files from selected branch on GitHub and puts them on your local PC. origin refers to Github, {branch} (usually main for main branch) refers to the branch you want to copy.
  • git checkout [branch name]: switches to the selected branch. Can be done with -b {branch name} to create and name a new branch, then switch to it.
  • git add [file name or -A for all]: stages the selcted changes to be added to a commit.
  • git commit: creates a version history snapshot.
  • git push: sends the commited (or snapshotted) changes to GitHub for a pull request.

Javascript

  • The Javascript file is linked within the "body" element.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
